# 인스펙터 케이스 테스트

## 범위
- 트랙 선택에 따른 인스펙터 패널 동기화
- 이펙트 추가/삭제 및 토글
- 패널 표시 및 리사이즈

## 환경
- 브라우저: Chromium 계열, Firefox
- OS: Windows
- 앱: Vite 개발 서버

## 테스트 데이터
- 트랙 2개 이상이 있는 프로젝트
- 인스펙터 패널이 보이는 상태

## 테스트 케이스

### IN-001 선택 트랙 동기화
사전조건: 트랙 리스트와 인스펙터 패널이 동시에 표시
절차:
1. 트랙 리스트에서 트랙 1을 선택한다
2. 인스펙터에 트랙 1 정보가 표시되는지 확인한다
기대결과:
- 인스펙터 내용이 선택한 트랙과 동기화된다

### IN-002 이펙트 추가/삭제
사전조건: 인스펙터 패널 표시 상태
절차:
1. 트랙 1에 이펙트를 추가한다
2. 추가한 이펙트를 삭제한다
기대결과:
- 이펙트가 추가되고 제거된다

### IN-003 이펙트 활성/비활성 토글
사전조건: 트랙에 이펙트가 1개 이상 존재
절차:
1. 이펙트를 비활성화한다
2. 이펙트를 다시 활성화한다
기대결과:
- 이펙트 활성 상태가 토글된다

### IN-006 이펙트 순서 드래그 앤 드롭
사전조건: 트랙에 이펙트가 2개 이상 존재
절차:
1. 이펙트를 드래그하여 다른 위치로 이동한다
기대결과:
- 이펙트 순서가 드래그한 위치로 변경된다

### IN-004 인스펙터 패널 표시 토글
사전조건: 툴바 표시 상태
절차:
1. 인스펙터 패널 토글을 꺼서 숨긴다
2. 다시 켜서 표시한다
기대결과:
- 인스펙터 패널이 정상적으로 숨김/표시된다

### IN-005 인스펙터 패널 리사이즈
사전조건: 인스펙터 패널 표시 상태
절차:
1. 인스펙터 리사이즈 핸들을 드래그해 너비를 늘린다
2. 다시 드래그해 너비를 줄인다
기대결과:
- 최소/최대 범위 내에서 너비가 부드럽게 변경된다
