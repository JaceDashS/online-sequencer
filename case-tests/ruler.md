# 룰러 케이스 테스트

## 범위
- 마디/비트 마커 표시
- 플레이헤드 이동/드래그
- Export 범위/로케이터 조작
- 수평 스크롤 동기화

## 환경
- 브라우저: Chromium 계열, Firefox
- OS: Windows
- 앱: Vite 개발 서버

## 테스트 데이터
- 타임라인과 룰러가 보이는 상태
- MIDI 파트가 있는 프로젝트

## 테스트 케이스

### RU-001 마디/비트 마커 표시
사전조건: 룰러 표시 상태
절차:
1. 룰러에 마디 번호와 비트 마커가 표시되는지 확인한다
기대결과:
- 마디 번호와 비트 마커가 정상적으로 표시된다

### RU-002 BPM 변경 반영
사전조건: 룰러 표시 상태
절차:
1. BPM 값을 변경한다
2. 룰러 마커 간격이 변하는지 확인한다
기대결과:
- BPM 변경에 따라 마커 간격이 반영된다

### RU-003 박자표 변경 반영
사전조건: 룰러 표시 상태
절차:
1. 박자표 값을 변경한다
2. 마디/비트 마커 구성이 변경되는지 확인한다
기대결과:
- 박자표 변경에 따라 마커 구성이 반영된다

### RU-004 룰러 클릭으로 플레이헤드 이동
사전조건: 룰러 표시 상태
절차:
1. 룰러 영역을 클릭한다
기대결과:
- 플레이헤드가 클릭한 위치로 이동한다

### RU-005 플레이헤드 드래그 이동
사전조건: 플레이헤드가 보이는 상태
절차:
1. 플레이헤드를 드래그한다
기대결과:
- 플레이헤드가 드래그 위치로 이동한다

### RU-006 Export 범위 드래그 (Ctrl+Alt)
사전조건: 룰러 표시 상태
절차:
1. Ctrl+Alt를 누른 채 룰러를 드래그한다
기대결과:
- Export 범위가 생성되고 범위 오버레이가 표시된다

### RU-007 Left/Right 로케이터 설정 (Ctrl/Alt)
사전조건: 룰러 표시 상태
절차:
1. Ctrl+클릭으로 Left 로케이터를 설정한다
2. Alt+클릭으로 Right 로케이터를 설정한다
기대결과:
- Left/Right 로케이터가 각각 설정된다

### RU-008 로케이터 드래그 조정
사전조건: Export 범위가 설정된 상태
절차:
1. Left 로케이터 핸들을 드래그한다
2. Right 로케이터 핸들을 드래그한다
기대결과:
- Export 범위가 드래그에 따라 조정된다

### RU-009 수평 스크롤 동기화
사전조건: 수평 스크롤이 가능한 상태
절차:
1. 하단 스크롤바를 좌우로 스크롤한다
기대결과:
- 룰러 내용이 스크롤에 맞춰 이동한다
