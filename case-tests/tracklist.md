# 트랙 리스트 케이스 테스트

## 범위
- 트랙 리스트 패널 UI 및 리사이즈
- 트랙 추가/삭제
- 솔로/뮤트 규칙
- 볼륨/팬/악기 설정

## 환경
- 브라우저: Chromium 계열, Firefox
- OS: Windows
- 앱: Vite 개발 서버

## 테스트 데이터
- 트랙 3개가 있는 프로젝트
- 트랙 1: 짧은 MIDI 클립 여러 개
- 트랙 2: 긴 MIDI 클립 1개
- 트랙 3: 비어 있음

## 테스트 케이스

### TL-001 트랙 리스트 리사이즈
사전조건: 앱 로드 완료
절차:
1. 트랙 리스트 리사이즈 핸들을 드래그해 너비를 늘린다
2. 다시 드래그해 너비를 줄인다
기대결과:
- 최소/최대 범위 내에서 너비가 부드럽게 변경된다

### TL-002 트랙 추가
사전조건: 트랙 수가 6개 미만
절차:
1. 트랙 추가 버튼을 클릭한다
기대결과:
- 기본 설정의 새 트랙이 추가된다

### TL-003 트랙 삭제
사전조건: 트랙이 1개 이상 존재
절차:
1. 트랙의 삭제 버튼을 클릭한다
2. 확인 다이얼로그에서 확인한다
기대결과:
- 해당 트랙이 목록에서 제거된다
- 삭제한 트랙이 선택 상태였다면 선택이 해제된다

### TL-005 트랙 볼륨/팬 변경
사전조건: 트랙 1개 이상
절차:
1. 볼륨 슬라이더를 드래그하여 조정하고 라벨 변화를 확인한다
2. 팬 슬라이더를 드래그하여 조정하고 라벨 변화를 확인한다
3. 볼륨 슬라이더 위에서 마우스 휠을 사용하여 값을 변경한다
4. 팬 슬라이더를 더블클릭하여 기본값으로 리셋한다
기대결과:
- 슬라이더를 드래그하면 값이 즉시 업데이트되고 라벨이 반영된다
- 마우스 휠로도 볼륨 값이 변경된다
- 더블클릭으로 기본값으로 리셋된다
- 슬라이더가 부드럽게 움직이고 반응한다

### TL-006 악기 변경
사전조건: 트랙 1개 이상
절차:
1. 악기 셀렉트를 변경한다
기대결과:
- 선택한 악기가 트랙에 반영된다

### TL-004 트랙 솔로/뮤트 기본 규칙
사전조건: 트랙 2개 이상
절차:
1. 트랙 1의 Solo를 켠다
2. 다른 트랙이 솔로에 의해 뮤트되는지 확인한다
3. 트랙 1의 Solo를 끈다
기대결과:
- 다른 트랙이 이전 뮤트 상태로 복원된다

### TL-010 솔로 시 다른 트랙 자동 뮤트
사전조건: 트랙 1, 트랙 2 존재
절차:
1. 트랙 1의 Solo를 켠다
기대결과:
- 트랙 2가 자동 뮤트 상태가 된다

### TL-011 다중 솔로에서 솔로 해제 시 자동 뮤트
사전조건: 트랙 1, 트랙 2 존재
절차:
1. 트랙 1의 Solo를 켠다
2. 트랙 2의 Solo를 켠다
3. 트랙 1의 Solo를 끈다
기대결과:
- 트랙 1이 자동 뮤트 상태가 된다

### TL-008 뮤트 상태에서 솔로 활성화 후 해제
사전조건: 트랙 1이 뮤트 상태
절차:
1. 트랙 1의 Solo를 켠다
2. 트랙 1의 Solo를 끈다
기대결과:
- 트랙 1은 뮤트 상태로 유지된다

### TL-012 뮤트 상태가 솔로 해제 후에도 유지
사전조건: 트랙 1이 뮤트 상태, 트랙 2 존재
절차:
1. 트랙 2의 Solo를 켠다
2. 트랙 2의 Solo를 끈다
기대결과:
- 트랙 1의 뮤트 상태가 유지된다

### TL-009 솔로 상태에서 뮤트 토글 후 해제
사전조건: 트랙 1이 중립 상태
절차:
1. 트랙 1의 Solo를 켠다
2. 트랙 1의 Mute를 켠다
3. 트랙 1의 Mute를 끈다
기대결과:
- 트랙 1은 중립 상태(솔로/뮤트 해제)로 돌아온다

### TL-013 솔로 상태에서 뮤트 시 자동 뮤트 해제
사전조건: 트랙 3개 이상, 트랙 2는 수동 뮤트 상태
절차:
1. 트랙 1의 Solo를 켠다
2. 트랙 3이 자동 뮤트 상태인지 확인한다
3. 트랙 1의 Mute를 누른다
기대결과:
- 솔로로 인해 자동으로 생긴 뮤트는 해제된다
- 사용자가 직접 뮤트한 트랙의 뮤트는 유지된다

### TL-014 트랙 추가 시 트랙 컨트롤러 즉시 표시
사전조건: 트랙 리스트 패널이 표시된 상태, 트랙 수가 6개 미만
절차:
1. 트랙 추가 버튼(+)을 클릭한다
2. 트랙 리스트에 새 트랙 컨트롤러가 즉시 표시되는지 확인한다
3. 새 트랙의 이름, 솔로/뮤트 버튼, 볼륨/팬 컨트롤이 모두 보이는지 확인한다
기대결과:
- 트랙 추가 버튼 클릭 직후 새 트랙 컨트롤러가 트랙 리스트에 표시된다
- 새 트랙의 모든 UI 요소(이름, 버튼, 컨트롤)가 정상적으로 렌더링된다
- 리렌더링 지연이나 UI 업데이트 누락 없이 즉시 반영된다
