# 툴바 케이스 테스트

## 범위
- BPM 및 박자표 변경
- Quantize/Metronome 토글
- Split/Merge 버튼 토글
- Zoom 및 패널 토글
- 녹음 토글 및 테마 전환

## 환경
- 브라우저: Chromium 계열, Firefox
- OS: Windows
- 앱: Vite 개발 서버

## 테스트 데이터
- MIDI 파트가 있는 프로젝트
- 타임라인/트랙 리스트가 보이는 상태

## 테스트 케이스

### TB-001 재생/정지 버튼 (클릭/단축키)
사전조건: 툴바 표시 상태
절차:
1. 재생 버튼을 클릭한다
2. 정지 버튼을 클릭한다
3. 재생/정지 단축키로 동일 동작이 되는지 확인한다
기대결과:
- 재생과 정지가 클릭/단축키 모두로 동작한다

### TB-011 플레이헤드 부드러운 재생
사전조건: 재생 가능한 프로젝트 상태
절차:
1. 재생 버튼을 클릭하여 재생을 시작한다
2. 플레이헤드가 부드럽게 이동하는지 관찰한다 (약 10-20초)
3. 플레이헤드가 박자 경계에서 멈추거나 떨리는 현상이 없는지 확인한다
기대결과:
- 플레이헤드가 일정하게 부드럽게 이동한다
- 박자 경계에서 멈추거나 떨리는 현상이 발생하지 않는다
- 재생 시간이 정확하게 진행된다

### TB-002 녹음 버튼 (클릭/단축키)
사전조건: 툴바 표시 상태
절차:
1. 녹음 버튼을 클릭한다
2. 녹음 버튼을 다시 클릭한다
3. 녹음 단축키로 동일 동작이 되는지 확인한다
기대결과:
- 녹음 상태가 클릭/단축키 모두로 토글된다

### TB-003 Quantize 버튼 (클릭/단축키)
사전조건: 툴바 표시 상태
절차:
1. Quantize 버튼을 클릭한다
2. Quantize 버튼을 다시 클릭한다
3. Q 키로 동일 동작이 되는지 확인한다
기대결과:
- Quantize 상태가 클릭/단축키 모두로 토글된다

### TB-004 Metronome 버튼 (클릭/단축키)
사전조건: 툴바 표시 상태
절차:
1. Metronome 버튼을 클릭한다
2. Metronome 버튼을 다시 클릭한다
3. M 키로 동일 동작이 되는지 확인한다
기대결과:
- Metronome 상태가 클릭/단축키 모두로 토글된다

### TB-005 박자표 변경 (드롭다운/스크롤)
사전조건: 툴바 표시 상태
절차:
1. 박자표 드롭다운으로 값을 변경한다
2. 박자표 스크롤 입력으로 값을 변경한다
기대결과:
- 박자표가 변경되고 마디 룰러에 반영된다

### TB-006 BPM 변경 (타이핑/스크롤/드래그)
사전조건: 툴바 표시 상태
절차:
1. BPM 입력을 타이핑으로 변경한다
2. BPM 입력을 스크롤로 변경한다
3. BPM 입력을 드래그로 변경한다
기대결과:
- BPM이 변경되고 마디 룰러에 반영된다

### TB-007 Split/Merge 버튼 활성화
사전조건: MIDI 파트가 존재
절차:
1. Split 버튼을 눌러 활성화한다
2. Merge 버튼을 눌러 활성화한다
기대결과:
- Split/Merge가 각각 활성화된다

### TB-008 Zoom 컨트롤
사전조건: 타임라인 표시 상태
절차:
1. Zoom 컨트롤 값을 올린다
2. Zoom 컨트롤 값을 내린다
기대결과:
- 타임라인 가로 스케일이 변경된다

### TB-009 패널 토글 버튼
사전조건: 트랙 리스트/인스펙터/믹서 표시 상태
절차:
1. 트랙 리스트 토글을 끈다
2. 인스펙터 토글을 끈다
3. 믹서 토글을 끈다
4. 각 토글을 다시 켠다
기대결과:
- 각 패널이 정상적으로 숨김/표시된다

### TB-010 테마 토글
사전조건: 툴바 표시 상태
절차:
1. 테마 토글을 누른다
2. 테마 토글을 다시 누른다
기대결과:
- 테마가 전환된다
